TOP 5 MEMORY CONSUMERS & SOLUTIONS
═══════════════════════════════════════════════════════════════════════════════

1. Event Data (40-50%)
   • Consumers: Activity strings, UUIDs, timestamps
   • Solution: StringIntern
   • Impact: 99% reduction on activity storage

2. Event Attributes (25-30%)
   • Consumers: BTreeMap overhead per event
   • Solution: CompactAttributes with Arc<>
   • Impact: 5-10x deduplication

3. Trace Structure (10-15%)
   • Consumers: Vec<Event>, metadata, padding
   • Solution: Arc<Trace> for sharing
   • Impact: Incremental sharing benefit

4. PetriNet Structure (5-10%)
   • Consumers: Places, transitions, arc connectivity
   • Solution: AdjacencyLists + index-based traversal
   • Impact: 20% reduction + cache locality

5. Temporary Results (5%)
   • Consumers: Intermediate collections (DFG, stats)
   • Solution: Streaming/incremental computation
   • Impact: 25-30% reduction
